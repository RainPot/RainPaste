# RainPaste 键盘选中交互设计文档

**日期：** 2026-03-12

## 目标

为命令面板增加更高效的键盘导航能力：默认选中第一条历史，支持 `↑ / ↓` 切换当前选中项，并在按下 `Enter` 时直接复制当前选中内容。

## 交互规则

- 面板每次打开时，默认选中当前结果列表的第一条。
- 搜索词变化后：
  - 有结果时，选中重置为第一条。
  - 无结果时，取消选中。
- 按 `↓` 时，选中移动到下一条。
- 按 `↑` 时，选中移动到上一条。
- 按 `Enter` 时，立即复制当前选中项。
- 按 `Esc` 时，关闭面板。
- 鼠标点击某条历史时，也同步更新当前选中项。

## 技术方案

1. `AppViewModel`
   - 增加 `selectedItemID`
   - 增加默认选中、选择同步、上下移动、激活当前项等方法

2. `CommandPanelWindow`
   - 在窗口层拦截 `↑ / ↓ / Enter / Esc`
   - 将事件分发给视图模型，避免搜索框焦点吞掉按键

3. `MainWindowView`
   - 使用 `ScrollViewReader` 在选中项变化时滚动到可见区域
   - 将当前选中项高亮展示

## 风险

- 搜索框仍然保持焦点，窗口层拦截需要优先于 `TextField` 默认行为。
- 删除或过滤后若当前选中项消失，必须回退到第一条，否则回车会失效。
