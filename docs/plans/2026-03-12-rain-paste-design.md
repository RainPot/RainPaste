# RainPaste 设计文档

**日期：** 2026-03-12

## 目标

构建一个原生 macOS 粘贴板工具，采用 `SwiftUI + AppKit 桥接`，提供菜单栏快速访问、全局快捷键呼出、主窗口管理、文本历史持久化等能力。产品风格为偏极客化的高对比深色界面，强调高频操作效率和可管理性。

## 范围

### 首版包含

- 常驻菜单栏图标
- 主窗口历史管理
- 仅支持文本剪贴板历史
- 全局快捷键呼出主界面
- 搜索、置顶、删除、清空
- 本地持久化保存历史
- 可配置最大历史数量

### 首版不包含

- 图片、文件、富文本
- 云同步
- 标签、分组、导出
- 多设备同步
- 复杂自动化规则

## 整体架构

应用拆分为以下模块：

1. `App` 与窗口层
   - 基于 `SwiftUI App` 组织主窗口与设置入口。
   - 使用 `WindowGroup` 或独立窗口控制器承载主管理界面。

2. `菜单栏桥接层`
   - 使用 `NSStatusItem` 提供常驻图标。
   - 菜单中展示最近历史、打开主窗口、暂停监听、退出应用等操作。

3. `快捷键与激活层`
   - 使用 `AppKit` 注册全局快捷键。
   - 负责激活应用、显示主窗口、聚焦搜索框。

4. `剪贴板监听层`
   - 基于 `NSPasteboard.general.changeCount` 定时轮询。
   - 检测文本变化后完成标准化、去重、截断、过滤。

5. `历史存储层`
   - 使用本地 JSON 文件持久化。
   - 存储字段包含 `id`、`content`、`createdAt`、`isPinned`、`sourceHint`。

6. `状态管理层`
   - 统一由 `ObservableObject` 管理历史、搜索词、设置项和窗口行为。

## 交互设计

### 菜单栏入口

- 点击菜单栏图标后，展开一个轻量菜单。
- 顶部显示应用名和状态。
- 中间显示最近若干条历史，单击即可重新写回剪贴板。
- 底部提供：
  - 打开主窗口
  - 暂停或恢复监听
  - 清空历史
  - 退出

### 主窗口

主窗口采用双栏或单栏增强列表布局：

- 顶部区域
  - 应用标题
  - 搜索框
  - 快捷键提示
  - 历史数量统计
- 中间历史列表
  - 每项展示文本首行、长度、时间、置顶状态
  - 支持点击复制
  - 支持悬浮显示操作按钮
- 右上操作区
  - 清空历史
  - 打开设置
- 底部状态栏
  - 监听状态
  - 当前最大历史数

### 设置区域

设置优先集成在主窗口的独立面板中，避免首版增加过多窗口复杂度。包含：

- 全局快捷键展示
- 最大历史条数设置
- 是否忽略重复内容
- 是否在复制后自动关闭窗口

## 数据流设计

1. 轮询服务检测到 `changeCount` 变化。
2. 从系统剪贴板读取纯文本。
3. 经过清洗逻辑：
   - 去掉空字符串
   - 合并重复连续内容
   - 限制单条最大长度用于展示
4. 生成 `ClipboardItem` 模型并写入内存状态。
5. 状态层按规则重排：
   - 置顶项固定在前
   - 其余项按时间倒序
   - 超出最大数量时裁剪尾部
6. 持久化层异步落盘。
7. 菜单栏和主窗口订阅状态刷新显示。
8. 用户点击某条历史时，将内容重新写入系统剪贴板，并更新最近使用时间。

## 数据模型

```swift
struct ClipboardItem: Identifiable, Codable, Equatable {
    let id: UUID
    let content: String
    let createdAt: Date
    var isPinned: Bool
}

struct AppSettings: Codable, Equatable {
    var maxHistoryCount: Int
    var ignoresConsecutiveDuplicates: Bool
    var closesWindowAfterCopy: Bool
}
```

## 异常处理

首版以稳定为先，采用可恢复错误策略：

- 剪贴板读取失败
  - 忽略本次变化，继续监听
- 持久化文件损坏
  - 记录错误日志
  - 回退为空历史和默认设置
- 快捷键注册失败
  - 在主窗口显示提示
  - 保留菜单栏可用
- 历史过长导致界面卡顿
  - 展示层截断预览文本
  - 存储层限制总条数

## 视觉设计

界面风格采用“极客风控制台”取向，但保持原生审美：

- 背景：深石墨灰与轻微层次渐变
- 强调色：荧光绿与冷青色
- 字体：标题使用较有个性的等宽字体风格，正文使用系统字体以保证可读性
- 卡片：细描边、弱发光、高对比 hover
- 动效：窗口出现、列表 hover、搜索聚焦时提供短促反馈，不做冗余动画

## 测试策略

1. 单元测试
   - 文本清洗逻辑
   - 去重逻辑
   - 置顶排序逻辑
   - 历史裁剪逻辑
   - JSON 持久化读写

2. 集成验证
   - 启动后能读取已有持久化历史
   - 复制文本后历史能追加
   - 从主窗口点击条目后能重新写回系统剪贴板

3. 手工验证
   - 菜单栏入口可用
   - 快捷键可唤起窗口
   - 搜索、置顶、删除、清空流程正常
   - 重启后历史保留

## 实施建议

首版优先保证功能闭环和界面气质，再逐步增加快捷键配置编辑、更强的列表操作和更多数据类型支持。
